## CRITICAL BUGS

### 1. User::atlet() — Wrong Foreign Key
- **File:** `app/Models/User.php:53`
- **Code:** `return $this->hasMany(Atlet::class, 'id');`
- **Issue:** FK is `'id'` instead of `'user_id'`. Relationship is completely broken — returns wrong results everywhere.
- **Fix:** `return $this->hasMany(Atlet::class, 'user_id');`

### 2. Winner::certificate() — Lowercase Class Name
- **File:** `app/Models/Winner.php:38`
- **Code:** `belongsTo(certificate::class)`
- **Issue:** PHP class lookup is case-sensitive. `certificate::class` resolves to undefined class — fails at runtime.
- **Fix:** `belongsTo(Certificate::class)`

### 3. Pricing Calculation Logic Broken
- **File:** `app/Http/Controllers/PesertaController.php:83-101`
- **Issue:** Inside the `foreach ($sortedPricing as $pricing)` loop, the fallback price recalculates `$harga` on every iteration instead of only after the last matching tier. Result: final price is always based on the last pricing tier, not the correct one.
- **Fix:** Restructure loop — `break` after matching tier, move overflow calculation to after the loop.

## CRITICAL AUTHORIZATION

### 4. No Ownership Check on Atlet Delete
- **File:** `app/Http/Controllers/AtletController.php` — `destroy()`
- **Issue:** No verification that the atlet belongs to the authenticated user. User A can delete User B's atlets by guessing the ID.
- **Fix:** Add `abort_if($atlet->user_id !== auth()->id(), 403);`

### 5. No Ownership Check on Document Download
- **File:** `app/Http/Controllers/AtletController.php` — `downloadDocument($id)`
- **Issue:** Same as above — User A can download User B's uploaded documents.
- **Fix:** Verify `Atlet::findOrFail($id)->user_id === auth()->id()`

### 6. Payment Callback Has No User Ownership Check
- **File:** `app/Http/Controllers/PesertaController.php:216-266`
- **Issue:** Callback processes payment by order ID without verifying it belongs to the authenticated user. Attacker could manipulate other users' payment status.
- **Fix:** After fetching `$pembayaran`, check `$pembayaran->user_id === auth()->id()`

## CRITICAL SECURITY

### 7. Public API Routes — No Authentication
- **File:** `routes/api.php:21-22`
- **Routes:** `GET /api/kompetisi`, `GET /api/kompetisi/{id}`
- **Issue:** No auth middleware. All competition data is publicly accessible without login.
- **Fix:** Add `middleware('auth:sanctum')` — or confirm this is intentional public data.

### 8. Role Middleware Without Proper Role System
- **File:** `routes/web.php` — admin route group
- **Issue:** Uses `middleware('auth','role:admin')` but `User::role` is a plain string column with no dedicated role system or middleware class defined. If the `role` middleware class doesn't exist, Laravel will silently pass or throw a 500.
- **Fix:** Confirm the custom `role` middleware exists and correctly reads `users.role`.

## HIGH — N+1 QUERIES

### 9. KompetisiController::index() — PHP Sort Instead of DB
- **File:** `app/Http/Controllers/KompetisiController.php:21-33`
- **Code:** `Kompetisi::all()->sortByDesc("tutup_pendaftaran")`
- **Issue:** Loads ALL competitions into memory, sorts in PHP. Will crawl on large datasets.
- **Fix:** `Kompetisi::orderBy('tutup_pendaftaran', 'desc')->get()`

### 10. KompetisiController::kelompokUmur() — Double Query Same Record
- **File:** `app/Http/Controllers/KompetisiController.php:48-49`
- **Code:**
  ```php
  $nama_kompetisi = Kompetisi::find($id)->nama;
  $id_kompetisi   = Kompetisi::find($id)->id;
  ```
- **Fix:** `$kompetisi = Kompetisi::findOrFail($id);` then use `$kompetisi->nama` / `$kompetisi->id`

### 11. UnduhanController::showBukuAcara() — Nested Lazy Loading
- **File:** `app/Http/Controllers/UnduhanController.php:59-73`
- **Issue:** For each acara → for each participant → loads `trackRecords` separately. On 100 participants × 10 events = 1000+ queries.
- **Fix:** Eager load: `->with('pesertaSelesai.trackRecords')` when fetching acaras.

### 12. PesertaController::tagihan() — Unconfirmed Eager Load
- **File:** `app/Http/Controllers/PesertaController.php:87`
- **Code:** `$kompetisi->pricings->sortBy(...)` inside a loop
- **Issue:** If `pricings` is not eager-loaded before the loop, this fires a query per iteration.
- **Fix:** Confirm eager load or add `->load('pricings')` before loop.

## HIGH — VALIDATION GAPS

### 13. NIK Validates Length Only
- **File:** `app/Http/Controllers/AtletController.php:135`
- **Code:** `'nik' => 'required|string|size:16'`
- **Issue:** Accepts any 16-character string (e.g., `1111111111111111`). No format validation.
- **Fix:** Add regex for Indonesian NIK: `regex:/^\d{16}$/`

### 14. Phone Validation Has a Space Before `|`
- **File:** `app/Http/Controllers/ProfileController.php:40`
- **Code:** `'phone' => 'required|string|regex:/^(0)[8][1-9][0-9]{6,11}$/| max:15'`
- **Issue:** Space before `| max:15` — Laravel may not parse this rule correctly.
- **Fix:** Remove space: `...$/|max:15'`

### 15. No Min Bound on Integer Fields
- **File:** `app/Http/Controllers/KompetisiController.php:108-109`
- **Code:** `"additional_price" => "nullable|integer"`, `"max_participation" => "nullable|integer"`
- **Issue:** Accepts negative values.
- **Fix:** Add `|min:0`

## MEDIUM — BUSINESS LOGIC

### 16. Age Filtering — Year Only, Ignores Month/Day
- **File:** `app/Http/Controllers/AcaraController.php:61-84`
- **Issue:** Age eligibility check compares `Carbon::parse($atlet->umur)->year` only. An athlete born Dec 31 and one born Jan 1 of the same year are treated identically, but may have very different ages for the competition date.
- **Fix:** Compare full date or use `age()` / `diffInYears()` from competition date.

### 17. No Uniqueness Check on Winner Import
- **File:** `app/Http/Controllers/WinnerController.php:66-80`
- **Issue:** Excel import doesn't check if winner.kode already exists for this kompetisi_id. Re-importing inserts duplicates.
- **Fix:** Add `updateOrCreate` logic in `KejuaraanImport` or pre-validate.

### 18. Pending Participant Cleanup — Only 2 Days Window
- **File:** `app/Console/Commands/DeletePendingParticipants.php:36`
- **Issue:** Deletes unpaid participants 2 days after registration close. No grace period for payment issues. Should be configurable.
- **Fix:** Move `2` to a config value; consider extending to 7 days.

## MEDIUM — SECURITY

### 19. Payment Callback — No Replay Attack Protection
- **File:** `app/Http/Controllers/PesertaController.php:216-266`
- **Issue:** Validates Midtrans signature but doesn't check timestamp or idempotency. Old successful callbacks could theoretically be replayed.
- **Fix:** Check `$pembayaran->status !== 'Berhasil'` before processing (likely already done but verify).

### 20. File Upload — User-Supplied Filename
- **File:** `app/Http/Controllers/WinnerController.php:155`
- **Code:** `$file->store('dokumen_kejuaraan', 'public')`
- **Issue:** Default `store()` generates hash name, which is fine. But verify no other upload path uses `getClientOriginalName()`.
- **Fix:** Audit all upload paths and ensure `hashName()` or `store()` is used, never `getClientOriginalName()`.

### 21. No Rate Limiting on Payment Endpoints
- **File:** `routes/web.php` — `/dashboard/generate-snap-token`
- **Issue:** No throttle middleware. Anyone authenticated can spam Midtrans snap token generation.
- **Fix:** Add `middleware('throttle:10,1')` to payment routes.

### 22. Hardcoded Admin Email in Console Command
- **File:** `app/Console/Commands/KirimLaporanVerifikasi.php:17`
- **Issue:** Email address hardcoded. Cannot change without modifying source code.
- **Fix:** Move to `.env` as `ADMIN_NOTIFICATION_EMAIL`.

### 23. XSS Risk on API Responses
- **Issue:** Blade auto-escapes, but API routes returning raw model data may expose unsanitized HTML if ever rendered client-side.
- **Fix:** Use API Resources (`php artisan make:resource`) to control output shape.
